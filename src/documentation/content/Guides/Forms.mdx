## Labels and Help Text

![Labels](https://blobscdn.gitbook.com/v0/b/gitbook-28427.appspot.com/o/assets%2F-LFNym8ScnaWKWBQFWTw%2F-LG1fFqQHhO5GOKdxHQo%2F-LG1gZow3tUzk1HV7Aqr%2Flabel-help-text%402x.png?alt=media&token=3c03d617-2750-4a3c-b279-cfc0219f6472)

<InfoCard title="Do:" color="green">
  <ul>
    <li>Labels should use title case, e.g. "Number of Employees".</li>
    <li>Labels should be short and clear.</li>
    <li>Help text should be used sparingly.</li>
  </ul>
</InfoCard>


<InfoCard title="Don't:" color="red">
  <ul>
    <li>Don't use all caps for form labels, e.g. "COMPANY NAME"</li>
    <li>Don't use all lowercase text for form labels, e.g. "company name"</li>
  </ul>
</InfoCard>

## Input States

### Error State

![Error Input](https://blobscdn.gitbook.com/v0/b/gitbook-28427.appspot.com/o/assets%2F-LFNym8ScnaWKWBQFWTw%2F-LG1fFqQHhO5GOKdxHQo%2F-LG1hHioFOq6x5PVsnP8%2Ftext-input-error-state%402x.png?alt=media&token=9089f0fc-4e29-40be-96aa-69cfc7618fe6)

<InfoCard title="Do:" color="green">
  <ul>
    <li>Be clear and concise with error text.</li>
    <li>Use the simplest language possible.</li>
    <li>Offer a direct solution if possible and/or link to additional information (FAQ, etc)</li>
  </ul>
</InfoCard>

<InfoCard title="Don't:" color="red">
  <ul>
    <li>Don't use jargon or technical terms.</li>
    <li>Don't display error or status codes.</li>
  </ul>
</InfoCard>

## Placeholder Text

<InfoCard title="Do:" color="green">
  <ul>
    <li>Use placeholders in addition to labels to provide helpful context.</li>
    <li>For example, you could add the placeholder text "Acme Inc." for an input labeled Company Name.</li>
  </ul>
</InfoCard>

<InfoCard title="Don't:" color="red">
  <ul>
    <li>Use placeholders instead of a label.</li>
    <li>Use a placeholder if the label is clear and common. For example, no placeholder text should be used for an input labeled "Email Address".</li>
  </ul>
</InfoCard>
